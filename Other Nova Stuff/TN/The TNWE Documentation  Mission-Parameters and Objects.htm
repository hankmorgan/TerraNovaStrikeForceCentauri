*** Mission-Parameters and Objects ***





Getting started

After installation of TN and the editors all you have to do to get started is to select which mission-file you like to edit. Depending on the TN-version you're using here is a description of how to get started:

  • TN full version: Here is a list of all the mission-files. All mission-files have the name "MISS*.RES" and are located in the "DATA"-directory
    A quick example would be EDMIS DATA\MISS3.RES

  • TN demo 1: There is only one mission: "MISS1D.RES". It is located in the "DATA"-directory in your TN-directory.
    A quick example would be EDMIS DATA\MISS1D.RES

  • TN demo 2: There are three missions: "MISS2D.RES, MISS3D.RES, MISS4D.RES". They are located in the "MAPS"-directory in your TN-directory.
    A quick example would be EDMIS MAPS\MISS2D.RES



    After starting EDMIS:

    All available keys (a subset of a,b,..z,A,B,...Z) are displayed on the screen and are depending on the menu you're currently in.
    In this documentation all the keys you have to press to get a certain operation done are displayed in Typewriter-letters.
    For most parameters, EDMIS asks for a number. This number can be entered either as a decimal number (i.e. 27) or an hexadecimal number prefixed by 0x (i.e. decimal 27 would be 0x1a). Nevertheless most values are displayed as hexadecimal numbers.

    Menu 1: Mission Parameters


    On top of the screen there are listed the files which are combined with this mission to get a complete TN-mission: A TN-mission consists of a mission-file, a map-file and a sky-file. The mission-file contains the objects of the mission, i.e. the enemies, the buildings, the parameters of the mission (storm, rain, gravity...) and so on. The map-file contains the terrain and the trees. The sky-file contains the sky and parts of the palette. Inside the map-file there is a description which planet the mission is located on. Theoretically it is possible to locate a mission on planet 2 and have the sky from planet 0 (namly sky 0), but this will look very ugly (try it out and you will see!).
    You can change the MAP-file and the SKY-file by pressing a, respectivly b.

    c Hazing: This sets the distanz to the haze: Usual range: 60 to 100

    d Temperature: Common range -52°C (Operation Meteor Shower) to +70°C (Operation Buzzard). D allows you to enter the absolut value in kelvin.
    e Gravity: This specifies the gravity of the planet. There are quite a lot of values possible, but in original TN there are only four different values used for the desert planet 0x14cce, the earth-like one 0x1000, the icy one 0x0e666 and the moon 0x09998.

    f Stormcode: Determines wheter the sky (clouds) are whirling (value 1) or not (value 0).
    g WindDir: Sets the direction from where the wind is blowing. 0° is wind from the South, 90° is from the East, 180° is from the North and 270° is from the West.
    h Windspeed: This sets the speed of the movement of the clouds. Common range is 0x0000 (calm) to 0x00fa (stormy). Higher values will lead to twisters ;)

    i CloudHeight: Simply the heigth of the clouds. Common range: 0x380 to 0x500
    j HorizonHeigth: This value is somewhat twisted with CloudHeight: For a normal/goodlooking sky take the value of CloudHeight and add 0x80 to it to receive the value for HorzionHeigth.

    k WhetherType and l WheatherDensity:
  • clear (type 0) -> density 0x0000
  • snow (type 1) -> density 0x0023 to 0x0050
  • rain (type 2) -> density 0x0028 to 0x0050



    Menu 2: Mission Objects

    Each object in the game is described by a class and a subclass. Take a look into classes.txt, which is included in the TNWE-Release. This file lists all available objects in their classes. You can set the class and the subclass with the a and the A (shift-a) keys. The editor will display the actual object in plain ascii right besides the number of the class and the subclass.
    The coordinates of the object will be entered in an integer and a fractal value: The integer values x and y (b and c) describe the square (tile) the object is located on. The z-value (d) specifies its heigth above ground. The fractal values of x and y (B(shift-b) and C(shift-c)) specify where within a square the object is located. The same applies to the fractal value of z ( D (shift-D)).
    The theta-value describes the rotation of the object around the x-axis (e). The phi-value specifies the rotation around the y-axis (f) and the omega-value the rotation around the z-axis (a).
    The next block of values differs depending on the class of the object. For easy use, take it like this: If it's an enemy or an vehicle, apply the "default pirate" values. If it's something else, apply the "clear entry", which just sets the last to blocks of values to zero. A more precise describtion will follow later.
    Note: All values in this menu page are displayed as hexadecimal!
    Note: All coordinates are the same coordinates used in tnedit!


    Menu 3: Mission Groups

    The interesting points are: 0 - the drop point and 31 - the pickup point. To set the drop point, press - until you see "0: drop pt." or something similar. Then press a to set the x-coordinate and A (shift-a) to set the y-coordinate. Note: Although you can enter the values either in decimal (i.e. 17) or hexadecimal (i.e. 0x1a), all values will be displayed in hexadecimal! To set the the pickup point press + until you see "31: pickup point" (or something similar). Then press h and H (shift-h) to set the coordinates (like done with the droppoint already).
    Note: All coordinates are the same coordinates used in tnedit!



    Menu 4: Mission Map Description

    During the game, you can take a look at a map on which some of the mission-objectives are painted. For instance: There is a point marking a pirate outpost, the pickup point and command post locations. These markers are defined in this section.
    Value a specifies the type of the marker: text, rectangle or circle. Depending on the type there has to be specified the coordinates of the marker (x and y in the same coordinates, tnedit uses), maybe a radius (for the circle), maybe the length of the sides (for the rectangle) or the text.
    Note: All coordinates are the same coordinates used in tnedit!
